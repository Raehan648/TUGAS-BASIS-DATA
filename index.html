Catatan Belajar Basis Data

Harapan Saya dari Kuliah Basis Data

Kalau ngomong jujur, saya pengen banget ngerti cara kerja database. Soalnya sekarang hampir semua aplikasi pasti butuh tempat nyimpen data. Harapan saya, dari mata kuliah ini saya bisa ngerti dari nol: mulai dari bikin desain database, belajar ERD, sampai nulis query SQL dengan lancar. Pokoknya, pengennya ilmu ini bisa jadi bekal buat kerja nanti.

Apa Itu Basis Data?

Basis data itu sebenarnya kayak lemari arsip digital. Bedanya, kalau arsip manual susah dicari, kalau database tinggal klik langsung ketemu. Contoh paling gampang ada di aplikasi sehari-hari: Instagram, Tokopedia, atau sistem kampus. Semuanya jalan karena ada database yang nyimpen data pengguna, transaksi, dan lainnya.

Conceptual Database

Sebelum bikin database di komputer, kita butuh gambaran besarnya dulu. Itulah yang disebut Conceptual Database. Biasanya digambarkan pakai ERD (Entity Relationship Diagram). Misalnya di kampus ada entitas Mahasiswa, Dosen, dan Mata Kuliah. Hubungannya bisa “Mahasiswa mengambil Mata Kuliah” atau “Dosen mengajar Mata Kuliah”. Dengan model ini, kita lebih gampang paham alur data sebelum masuk tahap teknis.

Physical Model

Setelah konsepnya jelas, barulah dibuat versi nyata di DBMS. Nah, inilah yang disebut Physical Model. Semua entitas tadi diubah jadi tabel lengkap dengan kolom, tipe data, serta aturan kayak primary key dan foreign key. Contoh: tabel Mahasiswa berisi NIM, Nama, dan Alamat. Dari sinilah database bisa dipakai beneran di aplikasi.
